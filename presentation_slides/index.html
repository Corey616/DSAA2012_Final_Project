Problem Definition & Challenges

🎯 Task Objective

  • Input: Multi-panel text scripts with [SCENE-N] tags and <Name> labels
  • Output: 3-frame visually coherent, character-consistent, narratively continuous image sequences
  • Application: Storyboard generation, animation preview, educational visualization

⚡ Core Challenges

Challenge Impact
Character Consistency Single person scenes produce multiple people, missing characters
Costume Drift Color changes across frames (blue shirt → green → dark blue)
Background Stability Scene element mutations, spatial-temporal chaos
VRAM Efficiency Panel 2+ OOM, model repeated loading

🔧 Technical Bottlenecks

SDXL Token Limit

CLIP encoding limit ~77 tokens (approx. 350 characters)

VAE Border Artifacts

SDXL VAE decoder generates gray/white borders

LLM Parsing Ambiguity

Pronoun resolution (He/She → specific character)

Multi-Character Synthesis

Spatial layout and identity preservation for 2+ characters

Innovation Contributions

🧮 Methodological Innovations

1. Description-Prefix Deduplication

Based on visual_description first 40 characters matching. Solves LLM case-variant character ambiguity (Case 04 "Two Milo" problem).

Dedup LLM Fix

2. Dynamic Weight Controller

Adjusts IP-Adapter/ControlNet weights based on action complexity. Balances "pose freedom" vs "identity consistency".

Adaptive SceneWeightController

3. Timeline Enforcement Engine

Multi-round statistical analysis determines dominant time_of_day. Automatically corrects time-shifted panels.

Consistency Time-of-Day

⚙ Engineering Innovations

4. Fully Offline Deployable

LLM (SDXL + Qwen) all run locally. No external API calls. Privacy-preserving + network-independent.

Local LLM Offline

5. 4-bit Quantization + VRAM Optimization

Qwen 4-bit quantization (4GB VRAM). SDXL FP16 + CPU Offload. Single GPU supports 3-frame generation.

NF4 FP16 CPU Offload

6. Unified Model Cache Management

./models/ centralized HuggingFace model management. Avoids repeated downloads, improves loading speed.

Cache HF Mirror
