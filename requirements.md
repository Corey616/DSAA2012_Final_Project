# Project 2: Multi‑Image Creative Generation

## Task 1: Story (Multi‑Shot / Multi‑Panel Generation)

Generate a sequence of images from a multi-panel text description.

**Important:** This is a choose‑one‑of‑two project. Each team should complete either Task 1 or Task 2, not both.  
*(This document focuses on Task 1.)*

## Provided Materials

The TA team will provide:

- a released development test set (Test‑A),
- a hidden holdout test set (Test‑B) for grading,
- input and output format specifications,
- a submission template,
- starter code for loading inputs and packaging outputs.

**Data collection is part of the project.**  
Each team must find, select, and prepare its own training/finetuning data (if needed) for the chosen task. You may use public datasets, self‑collected data, or other permissible external resources, provided that they comply with course policy and are fully disclosed in your report.

## Important Rules

This project has strict constraints:

- You may use Test‑A for development and reporting only.
- Your final system must generalize to the hidden Test‑B.
- Manual per‑case editing is not allowed.
- Hard‑coding outputs for specific test cases is not allowed.
- Agent‑based systems are not allowed and no external API usage is allowed.
- Each team is responsible for obtaining its own training data or external resources.
- Any external resources (pretrained models, public datasets, third‑party tools, etc.) must comply with course policy and must be clearly disclosed.
- Your submission must be a fixed, automatic, and reproducible pipeline.

## Task 1: Story

### Goal

Given a sequence of text descriptions (shots/panels), generate a corresponding sequence of images.

### Your outputs should satisfy

- **Per‑panel correctness:** each image matches its panel description.
- **Cross‑panel consistency:** recurring characters, objects, background, and style remain consistent across the sequence.
- **Narrative continuity:** the story progresses naturally from one panel to the next.

### Output

- Generate the required number of images for each test case.
- Follow the official file naming and resolution requirements.

### Evaluation (Task 1)

Your project will be evaluated primarily on the hidden Test‑B. Grading will be based mainly on human evaluation by the TAs and the instructor. We care about the relative quality of submissions, not an absolute automatic score.

**Evaluation focus for Story:**
- prompt adherence for each panel,
- narrative continuity,
- cross‑panel consistency,
- style consistency,
- image quality.
